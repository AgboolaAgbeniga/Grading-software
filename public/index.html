Grading Methodology

The HNG Grading Engine uses a deterministic verification system. We don't just check if a site is "online"; we perform a deep structural audit of both the deployed DOM and the underlying source code.

1. The 10-Point Scale

Each task (1A and 1B) is assigned a raw score out of 10. This is calculated based on the percentage of mandatory technical identifiers (data-testid) discovered by our crawlers:
(Unique IDs Found / Total Required IDs) × 10

2. Task Breakdown

Task 1A: Todo App (12 IDs)

Focuses on state management and CRUD interactivity. We check for form structures, input bindings, priority indicators, and overdue logic.

Task 1B: Profile Card (8 IDs)

Focuses on UI components and dynamic data rendering. We verify user identity elements, UTC time displays, and social link structures.

3. Dual-Channel Verification

To ensure academic integrity, we verify identifiers through two channels:

  • Live Scan: Our engine parses the rendered HTML on the deployed URL to ensure attributes exist in the final build.
  • Repo Audit: We recursively scan the GitHub repository's tree (HTML, JSX, TSX, JS, CSS) to verify the source code contains the required tags.

4. Integrity & Anti-Cheat

To maintain high standards, we enforce strict accessibility rules:

  • Broken Links: Any live URL that does not return a 200 OK status is automatically flagged as a FAIL.
  • Inaccessible Repos: Private repositories or 404 links that prevent source code audit are flagged as FAIL.
  • Requirement: Valid links are a prerequisite for scoring. An intern cannot pass if their work cannot be audited by the engine.

5. Pass Criteria

The final result is based on the Average Score of both tasks. An average of 7.0 or higher is required for a PASS status. Anything below triggers a REVIEW status, requiring manual intervention or improvement by the intern.
